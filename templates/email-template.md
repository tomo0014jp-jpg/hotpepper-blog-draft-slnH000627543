# email-template.md — メール件名・本文テンプレート（Gmail コネクター用）

> 1日3本のブログ下書きを、**1通のメール**にまとめて送付するためのテンプレートです。
> 本リポジトリではメール送信を **Gmail コネクター（MCP）の `send_email` ツール**を使って行います。
> Claude が組み立てた `subject` と `body` を、このツールの該当パラメータに渡してください。
>
> 詳細な送信フローは `CLAUDE.md` §6 を参照。

---

## このテンプレートが対応する `send_email` パラメータ

| `send_email` のパラメータ | このテンプレの対応箇所 | 備考 |
| --- | --- | --- |
| `to`      | `config/recipients.md` のモード対応の宛先 | TEST: `tomo0014jp@gmail.com` のみ |
| `subject` | 下記「件名（Subject）」の該当モードのフォーマット | TEST モードは `【TEST】` 必須 |
| `body`    | 下記「本文（Body）」のフォーマット | プレーンテキスト・絵文字なし |
| `cc` / `bcc` | （使わない） | 誤転送防止のため未指定 |

3本を別メールで送らない（= `send_email` 呼び出しも **1回のみ**）。

---

## 件名（Subject）

### TEST モード（`config/run-mode.md` が `TEST`）

```
【TEST】【ブログ下書き YYYY-MM-DD】{サロン名} 3本分
```

- **冒頭の `【TEST】` は必ず付けること。**
- `YYYY-MM-DD` は当日の日付（生成日）。
- `{サロン名}` は `config/salon.md` の値。本リポジトリでは `クラブオーサム(Club Awesome!)`。
  値が未取得なら `(サロン名未設定)` と入れる。
- 例: `【TEST】【ブログ下書き 2026-04-27】クラブオーサム(Club Awesome!) 3本分`

### PROD モード（`config/run-mode.md` が `PROD`）

```
【ブログ下書き YYYY-MM-DD】{サロン名} 3本分
```

- `【TEST】` は付けない。
- 例: `【ブログ下書き 2026-04-27】クラブオーサム(Club Awesome!) 3本分`

## 生成時のルール（Claude 用）

- **メールは必ず1通にまとめる**。3本を別メールで送らない（`send_email` 呼び出しも 1回のみ）。
- **TEST モードでは `subject` 冒頭に `【TEST】` を必ず付ける。**
- **TEST モードでは `body` 末尾に「※本メールはTEST送信です。お客様には送信されていません。」を必ず入れる。**
- **`to` フィールドは `config/run-mode.md` と `config/recipients.md` を必ずその場で参照して決定する。**
  - TEST モード時に PROD 宛先や任意の他アドレスを `to` に入れない。
  - 旧 Apps Script Webhook 方式と違い、サーバー側の宛先ガードはありません。Claude が唯一の砦です。
- 送付先は CC・BCC に複製しない（`cc` `bcc` は使わない）。
- 本文中に画像・画像URLを含めない。
- **`body` 内に絵文字を一切含めない（プレーンテキスト送信のため）。**
  - NG: ✨ 🌿 🌸 ☆ 🌟 ♪ ❤ 💕 😊 など Unicode の絵文字。Gmail のプレーンテキスト表示で文字化けします。
  - OK: `subject` 内の `【TEST】` `【ブログ下書き ...】` などの **括弧記号**。
  - OK: `body` 内の `────`（罫線）・`■`・`▼`・`／` などの日本語標準記号。
  - OK: ハッシュタグ案（`#xxx`）— 絵文字を含めなければ問題なし。
  - 季節感・親しみは語彙と文章のリズムで表現する。
- 各記事の `{...}` プレースホルダはすべて埋めること。空欄のまま送らない。
- `{宛名}` は PROD モードでは `config/recipients.md` の宛先表記、TEST モードでは「大石」とする。
